## **Lesson 2.3: Data Types**

### **The Three Main Categories:**

---

### **1. Structured Data**

**Definition:** Data organized in rows and columns (like Excel/database tables)

**Characteristics:**
- Fixed schema (predefined columns)
- Easy to search and query
- Fits neatly in databases

**Examples:**
```
Customer Table:
customer_id | name    | email              | age
101         | Ahmed   | ahmed@email.com    | 28
102         | Chioma  | chioma@email.com   | 34
```

**Where stored:** Relational databases (PostgreSQL, MySQL)

**How queried:** SQL

---

### **2. Semi-Structured Data**

**Definition:** Has some organization but not rigid rows/columns

**Characteristics:**
- Flexible schema
- Self-describing (contains metadata)
- Common in APIs and web data

**Examples:**

**JSON:**
```json
{
  "customer_id": 101,
  "name": "Ahmed",
  "email": "ahmed@email.com",
  "orders": [
    {"order_id": 5001, "total": 25000},
    {"order_id": 5002, "total": 12000}
  ]
}


{
  "customer_id": 101,
  "firstname": "Ahmed",
  "email": "ahmed@email.com",
  "orders_": [
    {"order_id": 5001, "total": 25000},
    {"order_id": 5002, "total": 12000}
  ]
}

```

**XML:**
```xml
<customer>
  <id>101</id>
  <name>Ahmed</name>
  <email>ahmed@email.com</email>
</customer>
```

**Where stored:** Data lakes, NoSQL databases (MongoDB)

**How queried:** Special parsers, can be converted to structured

---

### **3. Unstructured Data**

**Definition:** No predefined structure

**Characteristics:**
- Free-form
- Difficult to search directly
- Requires processing to extract meaning

**Examples:**
- Text documents (Word files, PDFs)
- Images (photos, scans)
- Videos
- Audio files
- Social media posts
- Email content

**Where stored:** Object storage (Amazon S3, minio), Data lakes.

**How processed:** AI/ML, text analysis, image recognition

---

### **Comparison Table:**

| Type | Structure | Example | Storage | Queryability |
|------|-----------|---------|---------|--------------|
| **Structured** | Rigid rows/columns | Database tables, CSV | Databases | Easy (SQL) |
| **Semi-Structured** | Flexible, nested | JSON, XML | Data lakes, NoSQL | Moderate |
| **Unstructured** | No structure | Images, PDFs, videos | Object storage | Hard (needs processing) |
